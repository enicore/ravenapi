9 * This class represents a form with HTML elements. It allows setting, validating, and extracting form data. The form
10 * handles elements (inputs, selects, etc.) and data (key-value pairs) for submission and validation.
11 *
12 * There are two main concepts:
13 * 1. Elements: A list of HTML form elements including their metadata (e.g., title, value, error state, etc.).
14 * 2. Data: A key-value array where each key corresponds to an element, and the value is the form input's data.
15 *
16 * The class provides methods to set form elements, validate the form data, handle errors, and extract data.
17 *
18 * There are functions to set elements and data. Data can be set from any key/value source, for example from the
19 * database. So initially we set up the elements and set the data with validation turned off. Then we send the elements
20 * to the frontend, render the form, and get the elements back - but with the value fields updated. Then we need to
21 * extract the data from that elements array (extractDataFromElements()) and set that data with validation turned on.
22 * This updates the elements' values and error fields which we can then send back to the frontend and save in the db.
